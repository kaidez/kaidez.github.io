---
name: review-post
description: Review a blog post markdown file for clarity, accuracy, SEO, readability, front matter consistency, and WCAG accessibility
---

Review the specified markdown file for:

1. **Clarity**: Flag unclear or ambiguous passages.

2. **Accuracy**: Check technical claims.

3. **SEO**: Evaluate title, headings, and keyword usage. For every `<a>` tag: check for meaningful link text (flag "click here" or bare URLs). For external links only (URLs starting with `http://` or `https://`): flag if `rel="noopener noreferrer"` is missing. Do not apply this check to internal links (paths starting with `/`). For every `<img>` tag: check that `alt` text is present and descriptive (not empty or filename-only), and that `width` and `height` attributes are set.

4. **Readability**: Evaluate readability at the sentence level. Flag sentences that feel genuinely dense or hard to follow — long sentences (roughly 20+ words) are worth examining, but use judgment rather than a hard cutoff. Not every long sentence is a problem. Compute a Flesch-Kincaid estimate.

5. **Front matter consistency**: Read the `title` and `excerpt` fields (the `excerpt` field maps to the page's meta description tag). Check two things:
   - That both fields exist and are well-formed
   - That both accurately reflect the current content of the post — flag any mismatch as a suggested update, especially if the body has been edited since they were written

   Length rules:
   - `title`: Soft warning if 61 characters or more (not a failure, just flag it)
   - `excerpt`: Strict limit of 140 characters — flag as an error if exceeded

6. **Accessibility (WCAG 2.1 AA)**: Check what can be verified from the markdown source.
   Note: this is not a full WCAG audit — checks requiring a rendered page (color contrast, focus indicators, ARIA roles) are out of scope here.

   What to check in the markdown:
   - Heading hierarchy: the `<h1>` is generated by Eleventy from the front matter `title`, so the markdown body should start at `##` (h2). Flag any use of `#` in the body, and flag any skipped heading levels (e.g., h2 → h4 with no h3)
   - Images: alt text is present and descriptive — decorative images should use `alt=""`
   - Links: link text is meaningful in isolation, without relying on surrounding context
   - Tables (if any): confirm a header row is defined

7. Provide specific rewrite or fix suggestions for each flagged item.
